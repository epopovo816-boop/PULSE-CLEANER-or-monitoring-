# Contributing to PULSE

Thank you for your interest in contributing to PULSE! We welcome all contributions, whether it's bug fixes, new features, documentation improvements, or translations.

## 📋 Code of Conduct

Please be respectful and constructive in all interactions. We aim to create a welcoming and inclusive community.

## 🚀 Getting Started

### 1. Fork & Clone
```bash
git clone https://github.com/YOUR-USERNAME/PULSE-CLEANER-or-monitoring-.git
cd PULSE-CLEANER-or-monitoring-
```

### 2. Create a Branch
```bash
git checkout -b feature/your-feature-name
```
or for bug fixes:
```bash
git checkout -b fix/bug-description
```

### 3. Make Changes
- Write clean, readable code
- Follow existing code style
- Test your changes thoroughly

### 4. Commit & Push
```bash
git add .
git commit -m "Clear description of changes"
git push origin your-branch-name
```

### 5. Create a Pull Request (PR)
- Go to the original repository
- Click "New Pull Request"
- Describe your changes clearly
- Reference any related Issues (#123)

## 📝 Guidelines

### Before You Start
- Check existing Issues and PRs to avoid duplicates
- For major changes, open an Issue first to discuss
- Read the README.md for project overview

### Code Standards
- Keep code clean and well-commented
- Follow the existing code style
- Test your changes before submitting
- Don't include unnecessary dependencies

### Commit Messages
- Use clear, descriptive messages
- Start with a verb: "Fix", "Add", "Update", "Remove"
- Example: "Add CPU temperature monitoring"

### Pull Request Checklist
- [ ] Changes follow code style
- [ ] All tests pass
- [ ] Documentation is updated if needed
- [ ] Commit messages are clear
- [ ] No breaking changes (unless discussed)

## 🐛 Reporting Bugs

Use the **Bug Report** template when creating an Issue:
- Clearly describe the problem
- Include steps to reproduce
- Show your environment (OS, PULSE version, RAM)
- Add screenshots if helpful

## ✨ Requesting Features

Use the **Feature Request** template:
- Explain the problem it solves
- Describe your proposed solution
- Suggest alternatives you've considered

## 🌍 Translations

Want to help translate PULSE?
- Add translations in the appropriate language files
- Include both UI strings and documentation
- Make sure translations are accurate and natural

## ❓ Questions?

- Open a **Question** Issue for help
- Check existing discussions first
- Be specific about your question
